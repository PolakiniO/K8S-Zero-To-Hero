# Recruiter Quick View (10–15 Minutes)

## Candidate signal summary

This repository demonstrates:
- production-style Kubernetes incident debugging,
- evidence-based root cause analysis,
- cloud architecture troubleshooting across EKS/EC2/S3,
- security risk modeling with misconfiguration chaining,
- clear communication from incident detection to prevention.

## What to look for in deliverables

- Clear symptom and impact statement.
- Objective evidence (events/logs/metrics/rollout history).
- Root cause backed by data, not guesswork.
- Safe mitigation + long-term corrective action.
- Preventive follow-up (alerts/runbooks/policy/tests).

## Interview prompts this repo can support

- "Walk me through a production outage you debugged."
- "How do you distinguish symptom vs root cause in Kubernetes incidents?"
- "How do you validate that a fix really solved the issue?"
- "How do small cloud misconfigurations combine into major risk?"
